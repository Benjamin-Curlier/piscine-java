# NIO.2 : Path et Files

## Pourquoi ce chapitre

Le chapitre 5-5 vous a montré l'I/O classique (entrées/sorties) : fonctionnelle, mais verbeux. Java 7 a introduit **NIO.2** (New I/O, version 2), une API (en anglais *Application Programming Interface*, interface de programmation) repensée qui simplifie considérablement la lecture et l'écriture de fichiers.

NIO.2 s'appuie sur deux classes centrales : `Path` (un chemin de fichier) et `Files` (les opérations sur ce chemin). Vous allez également voir comment combiner `Files.lines` avec les streams du module 4 — une occasion de relier deux acquis importants. **Les exercices I/O de la Piscine imposent NIO.2** : après ce chapitre, vous aurez tout ce qu'il faut.

## Ce que vous saurez faire à la fin

- **Créer** un `Path` avec `Path.of(...)` et expliquer sa différence avec `File`.
- **Lire** entièrement un fichier avec `Files.readAllLines` ou `Files.readString`.
- **Lire** un gros fichier ligne par ligne avec `Files.lines` en fermant correctement le flux (en anglais *stream*).
- **Écrire** dans un fichier avec `Files.writeString` ou `Files.write`.
- **Vérifier** l'existence d'un fichier et le copier avec `Files.exists` et `Files.copy`.
- **Spécifier** l'encodage UTF-8 (Universal Character Set Transformation Format, 8 bits) explicitement sur chaque opération.

## 1. Path : le chemin moderne

L'ancienne classe `File` (chapitre 5-5) décrit un chemin, mais ses méthodes renvoient souvent `false` sans explication quand une opération échoue. `Path` est son successeur : plus expressif, mieux intégré à l'API `Files`, et cohérent avec la gestion d'exceptions.

On crée un `Path` avec `Path.of(...)` (Java 11+) ou `Paths.get(...)` (Java 7+) — les deux formes sont équivalentes. Le chemin peut être relatif (depuis le répertoire courant) ou absolu.

### Exemple

```java
import java.nio.file.Path;

// Chemin relatif (depuis le répertoire d'exécution)
Path fichier = Path.of("donnees.txt");

// Chemin absolu (à éviter dans les exos — dépend de la machine)
Path absolu = Path.of("/tmp", "donnees.txt");

// Obtenir des informations sur le chemin (sans toucher au disque)
System.out.println(fichier.getFileName()); // donnees.txt
System.out.println(fichier.toAbsolutePath()); // chemin complet
```

### À retenir

> - `Path.of("fichier.txt")` crée un descripteur de chemin. Il ne crée pas le fichier.
> - Privilégiez `Path.of(...)` (plus lisible) à `Paths.get(...)`.
> - `Path` remplace `File` dans NIO.2 — il est conçu pour fonctionner avec la classe `Files`.

## 2. Lire un fichier

`Files` (dans le package `java.nio.file`) propose plusieurs méthodes de lecture selon la taille du fichier.

### Lire tout le contenu d'un coup (petit fichier)

`Files.readAllLines` charge toutes les lignes en mémoire d'un coup et les renvoie sous forme de `List<String>`. C'est pratique pour les petits fichiers. Pour les gros fichiers, préférez `Files.lines` (voir ci-dessous).

`Files.readString` charge tout le contenu en une seule `String` — utile quand vous traitez le texte en bloc.

### Lire ligne par ligne (gros fichier)

`Files.lines` renvoie un `Stream<String>` (flux au sens du module 4 — une séquence d'éléments traités à la demande). Ce `Stream` doit être **fermé** après utilisation, car il maintient le fichier ouvert en interne. On le place donc dans un `try-with-resources`.

### Exemple

```java
import java.io.IOException;
import java.nio.charset.StandardCharsets;
import java.nio.file.Files;
import java.nio.file.Path;
import java.util.List;
import java.util.stream.Stream;

Path fichier = Path.of("capitales.txt");

// Les méthodes de Files déclarent throws IOException (checked) : on les gère.
try {
    // Lecture complète en mémoire (petit fichier)
    List<String> lignes = Files.readAllLines(fichier, StandardCharsets.UTF_8);
    System.out.println("Nombre de capitales : " + lignes.size());

    // Lecture en flux : le Stream DOIT être fermé via try-with-resources
    try (Stream<String> flux = Files.lines(fichier, StandardCharsets.UTF_8)) {
        // on utilise les opérations de stream du module 4 directement sur le fichier
        long nombreAvecA = flux.filter(ligne -> ligne.contains("a")).count();
        System.out.println("Lignes contenant 'a' : " + nombreAvecA);
    }
} catch (IOException e) {
    System.out.println("Lecture impossible : " + e.getMessage());
}
```

### À retenir

> - `Files.readAllLines` : pratique pour les petits fichiers, tout est chargé en mémoire.
> - `Files.lines` : pour les gros fichiers — renvoie un `Stream` traité ligne par ligne. **Fermer le stream avec `try-with-resources`**, sans quoi le fichier reste ouvert.
> - Toujours préciser `StandardCharsets.UTF_8` en second paramètre pour éviter les accents cassés.

## 3. Écrire un fichier

`Files` propose deux méthodes d'écriture principales.

**`Files.writeString`** écrit une `String` entière dans le fichier, en créant ou en écrasant le fichier. C'est la méthode la plus directe pour du texte.

**`Files.write`** accepte une `List<String>` (ou un `Iterable<? extends CharSequence>`) : chaque élément devient une ligne.

Les deux méthodes écrasent le fichier par défaut. Pour ajouter en fin de fichier sans écraser, vous pouvez passer `java.nio.file.StandardOpenOption.APPEND` en paramètre supplémentaire.

### Exemple

```java
import java.io.IOException;
import java.nio.charset.StandardCharsets;
import java.nio.file.Files;
import java.nio.file.Path;
import java.util.List;

Path fichier = Path.of("fruits.txt");

// writeString et write déclarent throws IOException (checked) : on les gère.
try {
    // Écriture d'une chaîne multi-lignes
    Files.writeString(fichier, "pomme\npoire\nkiwi\n", StandardCharsets.UTF_8);

    // Écriture depuis une liste (chaque élément = une ligne)
    List<String> legumes = List.of("carotte", "poireau", "navet");
    Files.write(Path.of("legumes.txt"), legumes, StandardCharsets.UTF_8);
} catch (IOException e) {
    System.out.println("Écriture impossible : " + e.getMessage());
}
```

### À retenir

> - `Files.writeString` : écriture directe d'une `String` (avec `\n` pour les sauts de ligne).
> - `Files.write` : écriture depuis une `List<String>`, chaque élément sur une ligne.
> - Les deux méthodes **écrasent** le fichier existant par défaut. Ajoutez `StandardOpenOption.APPEND` pour ne pas écraser.

## 4. UTF-8 et utilitaires

Tous les exemples précédents précisent `StandardCharsets.UTF_8`. C'est une règle absolue dans ce module : ne jamais laisser Java choisir l'encodage par défaut. Sur Windows, l'encodage par défaut est souvent Windows-1252 ; sur Linux, UTF-8. Un fichier écrit sans encodage explicite sur Windows sera illisible sur Linux.

`Files` propose aussi des utilitaires pratiques :

```java
import java.io.IOException;
import java.nio.charset.StandardCharsets;
import java.nio.file.Files;
import java.nio.file.Path;

Path source = Path.of("fruits.txt");
Path destination = Path.of("fruits-sauvegarde.txt");

try {
    // Vérifier l'existence avant d'ouvrir (Files.size déclare throws IOException)
    if (Files.exists(source)) {
        System.out.println("Le fichier existe, taille : " + Files.size(source) + " octets");
    }

    // Copier un fichier
    Files.copy(source, destination);
    System.out.println("Copie réussie.");
} catch (IOException e) {
    System.out.println("Opération impossible : " + e.getMessage());
}
```

### Exemple complet — écrire, lire et transformer

Voici un enchaînement complet qui utilise `Files.writeString`, `Files.readAllLines` et `Files.lines` avec un stream pour illustrer les trois méthodes :

```java
import java.io.IOException;
import java.nio.charset.StandardCharsets;
import java.nio.file.Files;
import java.nio.file.Path;
import java.util.List;
import java.util.stream.Stream;

Path fichier = Path.of("fruits.txt");

// Tout le corps est gardé par un try/catch : les méthodes Files lèvent IOException.
try {
    Files.writeString(fichier, "pomme\npoire\nkiwi\n", StandardCharsets.UTF_8);

    // lecture simple, petit fichier
    List<String> lignes = Files.readAllLines(fichier, StandardCharsets.UTF_8);
    System.out.println("Lignes lues : " + lignes.size()); // 3

    // lecture en flux : le Stream DOIT être fermé
    try (Stream<String> flux = Files.lines(fichier, StandardCharsets.UTF_8)) {
        flux.map(String::toUpperCase).forEach(System.out::println);
        // POMME
        // POIRE
        // KIWI
    }
} catch (IOException e) {
    System.out.println("Erreur : " + e.getMessage());
}
```

### À retenir

> - `StandardCharsets.UTF_8` **partout** : à l'écriture et à la lecture, sur toutes les plateformes.
> - `Files.exists(path)` vérifie la présence d'un fichier sans exception.
> - `Files.copy(source, destination)` copie le contenu octet par octet.
> - Combiner `Files.lines(...)` avec les opérations de stream (`.filter`, `.map`, `.count`…) permet de traiter un fichier volumineux sans le charger entièrement en mémoire.

## Erreurs fréquentes

- **`Files.lines` sans `try-with-resources`** : écrire `Files.lines(fichier, StandardCharsets.UTF_8).forEach(...)` sans fermer le stream laisse le fichier ouvert jusqu'à la fin du programme. Cause : un `Stream` retourné par `Files.lines` maintient un descripteur de fichier ouvert. Correction : utiliser `try (Stream<String> flux = Files.lines(...)) { ... }`.

- **`Files.readAllLines` sur un fichier volumineux** : charger un journal de 2 Go avec `readAllLines` remplit la mémoire et provoque un `OutOfMemoryError`. Cause : tout le contenu est placé dans une `List` en mémoire vive. Correction : utiliser `Files.lines` (lecture en flux à la demande).

- **Encodage oublié** : appeler `Files.readAllLines(fichier)` sans préciser `StandardCharsets.UTF_8` utilise l'encodage par défaut de la JVM. Sur Linux en UTF-8, ça fonctionne. Sur Windows en Windows-1252, les accents deviennent illisibles. Correction : toujours passer `StandardCharsets.UTF_8` en second paramètre.

- **`Files.copy` sur un fichier existant** : `Files.copy(source, destination)` lève `FileAlreadyExistsException` si la destination existe déjà. Cause : comportement de sécurité par défaut. Correction : ajouter l'option `StandardCopyOption.REPLACE_EXISTING` si vous voulez écraser.

## Exercice guidé

**Contexte** : vous devez écrire un programme qui note une liste de fruits dans un fichier, puis relit ce fichier et affiche chaque fruit transformé en majuscules.

**Pas à pas :**

1. Créez un `Path` pointant vers `"fruits.txt"`.

2. Écrivez dans ce fichier la liste suivante avec `Files.writeString` : `pomme`, `poire`, `kiwi`, `mangue` (une par ligne, séparées par `\n`).

3. Ouvrez le fichier avec `Files.lines` dans un `try-with-resources` et enchaînez `.map(String::toUpperCase)` puis `.forEach(System.out::println)`.

4. Vérifiez que la sortie affiche `POMME`, `POIRE`, `KIWI`, `MANGUE`.

<details>
<summary>Solution (à n'ouvrir qu'après avoir essayé)</summary>

```java
import java.io.IOException;
import java.nio.charset.StandardCharsets;
import java.nio.file.Files;
import java.nio.file.Path;
import java.util.stream.Stream;

public class FruitsMajuscules {

    public static void main(String[] args) {

        Path fichier = Path.of("fruits.txt");

        // Étape 2 : écriture avec encodage UTF-8 explicite
        try {
            Files.writeString(fichier, "pomme\npoire\nkiwi\nmangue\n", StandardCharsets.UTF_8);
        } catch (IOException e) {
            System.out.println("Impossible d'écrire : " + e.getMessage());
            return;
        }

        // Étape 3 : lecture en flux, Stream fermé automatiquement
        try (Stream<String> flux = Files.lines(fichier, StandardCharsets.UTF_8)) {
            // map transforme chaque ligne en majuscules, forEach affiche
            flux.map(String::toUpperCase).forEach(System.out::println);
        } catch (IOException e) {
            System.out.println("Impossible de lire : " + e.getMessage());
        }
    }
}
```

Sortie attendue :
```text
POMME
POIRE
KIWI
MANGUE
```

**Points clés** :
- `Files.writeString` crée le fichier s'il n'existe pas.
- `Files.lines` + `try-with-resources` : le `Stream` est fermé dès la fin du bloc, même si une exception survient.
- `String::toUpperCase` est une référence de méthode (module 4) — elle remplace `ligne -> ligne.toUpperCase()`.

</details>
