## 2. Les 4 exercices

### 4.1.1 `annuaire-militaire` — gabarit — critère formateur `idiomatisme`
**Classe `Annuaire`** (champ interne `private final Map<String,String> annuaire = new HashMap<>();` — non observable).
```
void enregistrer(String indicatif, String nom)   // put
String rechercher(String indicatif)              // getOrDefault(indicatif, "Inconnu")
boolean supprimer(String indicatif)              // return annuaire.remove(indicatif) != null
int taille()                                     // size
List<String> indicatifsTries()                   // new ArrayList<>(keySet) trié alpha (Collections.sort ou stream().sorted())
```
**Starter** : `Annuaire` avec corps vides (`return "Inconnu";`/`return false;`/`return 0;`/`return List.of();` + `// TODO`).
**Tests publics** (≈5) : `enregistrer` puis `rechercher` renvoie le nom ; `rechercher` d'un absent → `"Inconnu"` ; réenregistrer un indicatif écrase l'ancien nom ; `taille` après 2 ajouts = 2 ; `indicatifsTries` → `containsExactly("ALPHA","BRAVO","CHARLIE")` (ordre alpha).
**Tests privés** (≈3) : `supprimer(absent)` → `false` (sans exception) ; annuaire vide → `taille` 0 et `indicatifsTries` vide ; `supprimer(présent)` → `true` puis `rechercher` → `"Inconnu"` et `taille` décrémentée.

### 4.1.2 `utilisateurs-uniques` — exo `equals`/`hashCode` à la main — critère formateur `idiomatisme`
**Classe `Utilisateur`** (champs `private final String nom; private final int matricule;` + constructeur + `getNom()`/`getMatricule()` ; **`equals`/`hashCode` à écrire**). Égalité logique = même `nom` ET même `matricule`.
**Classe `RegistreUtilisateurs`** (champ `private final Set<Utilisateur> utilisateurs = new HashSet<>();`).
```
boolean ajouter(Utilisateur u)    // return utilisateurs.add(u)  -> false si déjà présent (s'appuie sur equals/hashCode)
boolean contient(Utilisateur u)   // utilisateurs.contains(u)
int nombreDistincts()             // utilisateurs.size()
Set<Utilisateur> tous()           // new HashSet<>(utilisateurs) (copie défensive)
```
**Starter** : `Utilisateur` fournit champs + constructeur + accesseurs ; `equals`/`hashCode` en **TODO** (`@Override public boolean equals(Object o){ return false; // TODO }` et `@Override public int hashCode(){ return 0; // TODO }`). `RegistreUtilisateurs` stubs.
**Tests publics** (≈5) : deux `Utilisateur` de mêmes champs → `isEqualTo` ; champs différents → `isNotEqualTo` ; `a.equals(b)` ⇒ `a.hashCode() == b.hashCode()` ; `ajouter` d'un nouvel → `true`, d'un **doublon logique** (`new Utilisateur` égal) → `false` ; `nombreDistincts` ignore les doublons (ajouter 2 égaux → 1).
**Tests privés** (≈5, contrat gradué §10.2) : réflexivité (`a.equals(a)`) ; symétrie (`a.equals(b)` ET `b.equals(a)`) ; `a.equals(null)` → `false` ; `a.equals("texte")` (autre type) → `false` ; robustesse `HashSet` : ajouter 2 égaux + 1 différent → `nombreDistincts` = 2 et `contient(copieEgale)` → `true`. **NE PAS tester** transitivité, `getClass`-vs-`instanceof`, héritage.

### 4.1.3 `file-de-mission` — critère formateur `demarche`
**Classe `FileMission`** (champ `private final Deque<String> file = new ArrayDeque<>();`, FIFO).
```
void ajouter(String mission)      // addLast
String traiterProchaine()         // return file.pollFirst()  -> null si vide
String prochaine()                // return file.peekFirst()  -> null si vide (ne retire pas)
boolean estVide()                 // isEmpty
int taille()                      // size
```
**Tests publics** (≈4) : ordre FIFO (ajouter "A","B","C" → `traiterProchaine` renvoie "A" puis "B") ; `prochaine` renvoie la tête sans changer la taille ; `estVide` vrai au départ, faux après ajout ; `taille` correcte.
**Tests privés** (≈3) : `traiterProchaine` sur file vide → `null` (pas d'exception) ; `prochaine` sur file vide → `null` ; alternance ajout/retrait préserve l'ordre FIFO.

### 4.1.4 `groupement-par-grade` — critère formateur `demarche`
**Fourni dans le starter (`enum Grade`)** : `public enum Grade { SOLDAT, CAPORAL, SERGENT, ADJUDANT, LIEUTENANT }` (« FOURNI — ne pas modifier »).
**Classe `GroupesParGrade`** (champ `private final Map<Grade, List<String>> parGrade = new HashMap<>();`).
```
void affecter(Grade grade, String nom)   // parGrade.computeIfAbsent(grade, g -> new ArrayList<>()).add(nom)
List<String> membres(Grade grade)        // parGrade.getOrDefault(grade, List.of())  -> liste vide si jamais affecté
Set<Grade> grades()                      // parGrade.keySet()  (ou copie)
int effectif(Grade grade)                // membres(grade).size()
```
**Tests publics** (≈4) : affecter plusieurs noms au même grade → `membres` les accumule dans l'ordre (`containsExactly`) ; `grades()` = ensemble des grades affectés (`containsExactlyInAnyOrder`) ; `effectif` par grade ; un nom affecté 2× au même grade apparaît 2× (`List`, pas `Set`).
**Tests privés** (≈3) : `membres(grade jamais affecté)` → liste vide (pas `null`) ; `computeIfAbsent` ne réinitialise pas une liste existante (affecter A puis B au même grade → les deux présents) ; `effectif(grade jamais affecté)` → 0.
