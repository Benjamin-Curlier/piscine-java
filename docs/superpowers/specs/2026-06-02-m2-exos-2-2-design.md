## 1. Décisions tranchées au brainstorm (2026-06-02)

1. **Même approche que 2.1** : tout dans `main`, saisie clavier (`Scanner`), test sur la sortie standard (`CaptureEntree` + `CaptureSortie`). Le stagiaire n'écrit pas de méthode `static` (réservé à 2.3/2.4).
2. **Antériorité** : notions limitées au chapitre Chaînes ([`courses/docs/module-2-tableaux-chaines-methodes/2-3-chaines-de-caracteres.md`](../../../courses/docs/module-2-tableaux-chaines-methodes/2-3-chaines-de-caracteres.md)) : `length`, `charAt`, `substring`, `indexOf`, `equals`/`equalsIgnoreCase`, `toUpperCase`/`toLowerCase`, `trim`, `split`. Pas de `Character.isLetter`, pas de regex.
3. **Palindrome (2.2.1)** : entrée = un mot ; comparaison **insensible à la casse** ; sortie `oui` / `non`.
4. **Comptage (2.2.2)** : compte un **caractère**, **casse stricte** ; sortie `Occurrences : <n>`.
5. **ASCII art (2.2.3)** : triangle d'étoiles de hauteur `n` (ligne *i* = *i* étoiles `*`).
6. **Conventions de sortie figées** :
   - Palindrome → `oui` ou `non` (minuscules, sans accent).
   - Comptage → `Occurrences : <n>`.
7. **Difficulté / durée** : les 3 = `facile`. Durées : 25 / 20 / 20 min.

## 2. Conventions d'entrée (stdin)

- **2.2.1 palindrome** : une ligne contenant un mot (sans espace). Lue avec `nextLine()`, nettoyée avec `strip()` par sécurité.
- **2.2.2 comptage-occurrences** : ligne 1 = le texte (lu avec `nextLine()`, peut contenir des espaces) ; ligne 2 = le caractère (lu avec `nextLine()`, on prend `charAt(0)`).
- **2.2.3 ascii-art** : une ligne contenant l'entier `n` (lu avec `nextInt()`), `n ≥ 1`.

Les tests injectent l'entrée via `CaptureEntree`. Aucune entrée malformée n'est testée (cf. §8).

## 3. Spécification par exercice

### 2.2.1 — palindrome (`facile`, 25 min)
- **But pédagogique** : parcourir une chaîne par les deux bouts (ou la comparer à son envers), normaliser la casse.
- **Entrée** : un mot.
- **Traitement** : mettre le mot en minuscules (`toLowerCase`), puis vérifier qu'il se lit pareil à l'endroit et à l'envers.
- **Sortie** : `oui` si palindrome, sinon `non` (suivi d'un retour à la ligne).
- **Exemples** : `Radar` → `oui` ; `kayak` → `oui` ; `bonjour` → `non`.
- **Cas privés** : un seul caractère (`a` → `oui`) ; deux identiques (`aa` → `oui`) ; deux différents (`ab` → `non`) ; casse mixte (`Ressasser` → `oui`) ; mot non-palindrome de longueur paire (`abca` → `non`).

### 2.2.2 — comptage-occurrences (`facile`, 20 min)
- **But pédagogique** : parcourir une chaîne caractère par caractère (`charAt` sur `length`), compter.
- **Entrée** : ligne 1 = le texte ; ligne 2 = le caractère cherché.
- **Traitement** : compter, **en respectant la casse**, le nombre de positions où `charAt(i)` égale le caractère cherché.
- **Sortie** : `Occurrences : <n>`.
- **Exemples** : `mississippi` + `s` → `Occurrences : 4` ; `mississippi` + `i` → `Occurrences : 4`.
- **Cas privés** : caractère absent (`Occurrences : 0`) ; casse stricte (`Mississippi` + `M` → `1`, et + `m` → `0`) ; texte contenant des espaces (`a b a` + `a` → `2`) ; comptage d'un espace (`a b a` + espace → `2`).

### 2.2.3 — ascii-art (`facile`, 20 min)
- **But pédagogique** : double boucle, construction de lignes de longueur croissante.
- **Entrée** : un entier `n` (`n ≥ 1`).
- **Sortie** : `n` lignes ; la ligne *i* (de 1 à `n`) contient exactement *i* caractères `*`, suivie d'un retour à la ligne. Pas d'espace (que des étoiles) → comparaison exacte.
- **Exemple** : `n = 3` →
  ```
  *
  **
  ***
  ```
- **Cas privés** : `n = 1` (une ligne `*`) ; `n = 5` (pyramide rectangle de 5 lignes).

## 8. Hors périmètre (YAGNI)

- Pas de gestion d'entrée malformée (texte vide, `n` non entier) : l'énoncé garantit une entrée bien formée.
- Pas de palindrome de phrase (espaces/ponctuation) : entrée = un mot.
- Pas de comptage de sous-chaîne ni de regex : un seul caractère.
- Pas de méthodes `static` écrites par le stagiaire (réservé à 2.3/2.4).
