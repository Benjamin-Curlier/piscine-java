## 1. Décisions tranchées au brainstorm (2026-06-03)

1. **On teste des OBJETS** (instanciation + AssertJ), sans console.
2. **Squelette `starter/` « plus léger » mais qui COMPILE** (décision §10.1 pour 3.3) : le starter fournit la **classe de base** (abstraite ou concrète) **complète** et des **sous-classes pré-câblées** (déclaration `extends`, stub de constructeur, stubs `@Override` à corps par défaut). Le stagiaire **complète** : champs spécifiques, appel `super(...)`, corps des méthodes. Les tests publics compilent dès le départ ; la **conception** (quels champs, quel calcul polymorphe) reste à la charge du stagiaire. **Exception 3.3.4** : la hiérarchie `Forme` est fournie *entière* (on consomme le polymorphisme, on ne le construit pas) ; seule la classe utilitaire `Parc` est à écrire.
3. **`Personnel` est une classe ABSTRAITE** (les interfaces sont au sous-groupe 3.4 — antériorité). Idem toute base polymorphe ici : `abstract class` ou classe concrète, jamais `interface`.
4. **Vérifier le polymorphisme par compilation/comportement d'abord** (§10.2) : usage **via le type de base** (`Forme f = new Cercle(...); f.aire();`), redéfinition `@Override` observée par le comportement, méthode abstraite forcée par la compilation de la solution. **`instanceof` + downcast** testés en cas privé (3.3.4). Pas de réflexion (non nécessaire ici).
5. **Nommage** : membres en français, accesseurs `get` ; classes `PascalCase`. `toString`/`decrire`/`fiche` formatés avec `Locale.ROOT` dès qu'il y a des décimales.
6. **Difficulté / durée** : 3.3.1 `moyen` (35) / 3.3.2 `moyen` (30) / 3.3.3 `moyen` (35) / 3.3.4 `moyen` (35).

## 2. Spécification par exercice

Champs communs en `protected` dans les bases (idiome chapitre 3-7). Aucune exception. `prerequis: []`.

### 3.3.1 — forme (`moyen`, 35 min) — **gabarit** — classes abstraites
- **But** : classe abstraite avec méthodes **abstraites** (`aire`, `perimetre`) et méthode **concrète** (`decrire`) qui les appelle (polymorphisme / patron de méthode) ; sous-classes concrètes.
- **API publique imposée** :
  ```
  abstract class Forme
    abstract double aire()
    abstract double perimetre()
    String decrire()        // concret : "aire = %.2f, perimetre = %.2f" (Locale.ROOT)
  class Cercle extends Forme
    Cercle(double rayon)
    double getRayon()
    @Override double aire()       // Math.PI * rayon * rayon
    @Override double perimetre()  // 2 * Math.PI * rayon
  class Rectangle extends Forme
    Rectangle(double largeur, double hauteur)
    double getLargeur(), double getHauteur()
    @Override double aire()       // largeur * hauteur
    @Override double perimetre()  // 2 * (largeur + hauteur)
  ```
- **Contrat `decrire`** : `String.format(Locale.ROOT, "aire = %.2f, perimetre = %.2f", aire(), perimetre())`.
- **Starter** : `Forme` fournie complète (abstraites + `decrire` concret). `Cercle`/`Rectangle` pré-câblées (`extends Forme`, constructeur stub, `@Override` stubs `return 0;`). Le stagiaire ajoute les champs, `super` n'est pas requis (Forme n'a pas de constructeur), et les corps.
- **Cas publics** : `Cercle(2).aire()`/`perimetre()` (tolérance) ; `Rectangle(3,4).aire()`/`perimetre()` ; **usage polymorphe** `Forme f = new Cercle(1); f.aire()` ; `decrire()` d'un `Rectangle` au format attendu.
- **Cas privés** : `decrire()` appelé via le **type de base** `Forme` sur un `Cercle` (vérifie le bon `aire()` polymorphe) ; `Rectangle` carré (3,3) ; `Cercle(0)` → aire 0 ; un `Forme[]` mixte dont on lit `aire()` sur chaque élément.

### 3.3.2 — vehicule (`moyen`, 30 min) — héritage simple, `super`, `@Override`
- **But** : base **concrète**, sous-classes qui appellent `super(...)` et **redéfinissent** une méthode (en réutilisant `super.decrire()`).
- **API publique imposée** :
  ```
  class Vehicule
    Vehicule(String marque, int vitesseMax)
    String getMarque()
    int getVitesseMax()
    String decrire()        // "Vehicule <marque> (<vitesseMax> km/h)"
  class Voiture extends Vehicule
    Voiture(String marque, int vitesseMax, int nbPortes)
    int getNbPortes()
    @Override String decrire()   // "Voiture <marque> (<vitesseMax> km/h, <nbPortes> portes)"
  class Moto extends Vehicule
    Moto(String marque, int vitesseMax)
    @Override String decrire()   // "Moto <marque> (<vitesseMax> km/h)"
  ```
- **Champs** `marque`, `vitesseMax` en `protected` dans `Vehicule`. Les `@Override` peuvent réutiliser `super.decrire()` ou reformer la chaîne (le test ne vérifie que le résultat).
- **Starter** : `Vehicule` complète. `Voiture`/`Moto` pré-câblées (`extends Vehicule`, constructeur stub avec `super(...)` à compléter, `@Override decrire()` stub `return null;`).
- **Cas publics** : `Vehicule("Renault", 180).decrire()` ; `Voiture(...).decrire()` ; `Moto(...).decrire()` ; getters (dont `getNbPortes`).
- **Cas privés** : usage **polymorphe** `Vehicule v = new Voiture(...); v.decrire()` (rend la version Voiture) ; `getMarque()` hérité sur une `Moto` ; `Vehicule[]` mixte dont on appelle `decrire()` sur chaque élément.

### 3.3.3 — personnel (`moyen`, 35 min) — hiérarchie abstraite
- **But** : base **abstraite** avec deux méthodes abstraites (`grade`, `solde`) et une concrète (`fiche`) ; trois sous-classes spécialisées.
- **API publique imposée** :
  ```
  abstract class Personnel
    Personnel(String nom)
    String getNom()
    abstract String grade()
    abstract double solde()
    String fiche()          // "<nom> — <grade> — %.2f €" (Locale.ROOT)
  class Officier extends Personnel
    Officier(String nom, int echelon)     // solde = 3000 + 200 * echelon ; grade = "Officier"
  class SousOfficier extends Personnel
    SousOfficier(String nom, int anciennete)  // solde = 2200 + 100 * anciennete ; grade = "Sous-officier"
  class MilitaireDuRang extends Personnel
    MilitaireDuRang(String nom)            // solde = 1600 ; grade = "Militaire du rang"
  ```
- **Contrat `fiche`** : `String.format(Locale.ROOT, "%s — %s — %.2f €", getNom(), grade(), solde())`. Le séparateur est `" — "` (espace, tiret cadratin U+2014, espace), identique au format des chapitres.
- **Champ** `nom` en `protected` dans `Personnel`. Sous-classes : champ spécifique privé + `super(nom)`.
- **Starter** : `Personnel` complète (abstraites + `fiche`). `Officier`/`SousOfficier`/`MilitaireDuRang` pré-câblées (`extends Personnel`, constructeur stub avec `super(nom)`, `@Override grade()`/`solde()` stubs).
- **Cas publics** : `Officier("Durand", 3).grade()`/`solde()` ; `SousOfficier(...).solde()` ; `MilitaireDuRang("X").solde()` ; `fiche()` d'un `Officier`.
- **Cas privés** : `fiche()` via le type de base `Personnel` (polymorphe) ; `getNom()` hérité ; `Personnel[]` mixte dont on lit `solde()` ; un calcul de solde aux valeurs limites (échelon 0).

### 3.3.4 — parc (`moyen`, 35 min) — consommer le polymorphisme + `instanceof`
- **But** : écrire des opérations **polymorphes** sur un `Forme[]` ; usage de `instanceof` + downcast.
- **La hiérarchie `Forme`/`Cercle`/`Rectangle` est FOURNIE entière** (starter ET solution identiques pour ces 3 classes) — reprise de 3.3.1 (mêmes contrats). Le stagiaire n'écrit que `Parc`.
- **API publique imposée** (à écrire) :
  ```
  class Parc
    static double surfaceTotale(Forme[] formes)   // somme des aire()
    static Forme plusGrande(Forme[] formes)       // l'élément d'aire maximale (null si tableau vide)
    static int compterCercles(Forme[] formes)     // nombre d'éléments qui sont des Cercle (instanceof)
  ```
- **Starter** : `Forme`/`Cercle`/`Rectangle` complètes ; `Parc` avec les 3 signatures `static` stub (`return 0;` / `return null;`).
- **Cas publics** : `surfaceTotale` sur `{Cercle(1), Rectangle(2,3)}` ; `plusGrande` renvoie la bonne forme ; `compterCercles` compte les cercles.
- **Cas privés** : `surfaceTotale` d'un tableau vide → `0.0` ; `plusGrande` d'un tableau vide → `null` ; `compterCercles` sur un tableau sans cercle → `0` ; `plusGrande` quand le max est un `Rectangle` ; downcast après `instanceof` (ex. récupérer le `getRayon()` du `Cercle` renvoyé par `plusGrande`).

## 7. Hors périmètre (YAGNI)

- **Pas d'interface, record, enum, sealed** (sous-groupe 3.4).
- **Pas de collections/génériques/streams** (M4) : on utilise des **tableaux** (`Forme[]`).
- **Pas d'exceptions** (M5) : `plusGrande([])` renvoie `null` (documenté), pas de `throw`.
- **Pas d'`equals`/`hashCode` manuel** (M4).
- **Date/heure réelles, calculs financiers exacts** : hors sujet.
