## 2. Les 3 exercices

### 5.3.1 `import-csv-personnel` — CSV → `List<Personnel>` avec exceptions — critère formateur `demarche`

**Pont formats ↔ exceptions.** Classe `ImportCsv` (+ `Grade`/`Personnel` fournis) :
```
static List<Personnel> importer(Path csv) throws IOException
//   Files.readAllLines(csv, UTF_8) ; SAUTER l'en-tête (1re ligne "nom,grade,anciennete") ;
//   IGNORER les lignes blanches ; pour chaque ligne de données : split(",") en 3 champs :
//     - champs.length != 3            -> throw IllegalArgumentException("ligne invalide ... : <ligne>")
//     - Grade.valueOf(champs[1].trim()) échoue (IllegalArgumentException) -> rethrow
//          IllegalArgumentException("grade inconnu : <g> (ligne : <ligne>)", cause)   // CHAÎNAGE
//     - Integer.parseInt(champs[2].trim()) échoue (NumberFormatException) -> rethrow
//          IllegalArgumentException("anciennete invalide : <a> (ligne : <ligne>)", cause)  // CHAÎNAGE
//   sinon ajoute new Personnel(champs[0].trim(), grade, anciennete) à la liste.
```
Démarche (critère) : structurer la boucle (saut d'en-tête, validation du nombre de champs, conversions gardées, **chaînage de la cause**), accumuler la liste. **Toutes** les lignes invalides → `IllegalArgumentException` (cause chaînée pour grade/ancienneté). **Starter** : `return List.of(); // TODO`.

**Publics** (≈8) : CSV valide 3 lignes → `containsExactly` les 3 `Personnel` attendus ; en-tête sauté (pas de `Personnel` « nom,grade,anciennete ») ; lignes blanches ignorées ; fichier en-tête seul → liste vide ; grade inconnu (`"Dupont,GENERAL,5"`) → `IllegalArgumentException` ; ancienneté non numérique (`"Dupont,SERGENT,abc"`) → `IllegalArgumentException` ; nombre de champs ≠ 3 (`"Dupont,SERGENT"`) → `IllegalArgumentException` ; message contient l'info fautive.
**Privés** (≈8) : grade inconnu → `getCause()` instanceof `IllegalArgumentException` (celle de `Grade.valueOf`) ; ancienneté non numérique → `getCause()` instanceof `NumberFormatException` ; trim des champs (`" Dupont , SERGENT , 5 "` → `Personnel("Dupont", SERGENT, 5)`) ; accents UTF-8 dans le nom ; ordre préservé ; 1 seule ligne de données ; ligne à 4 champs → exception ; ancienneté négative acceptée (pas de garde métier ici, on lit ce qui est écrit).

### 5.3.2 `export-csv-trie` — tri (`Comparator`) + écriture CSV — critère formateur `idiomatisme`

Classe `ExportCsv` (+ `Grade`/`Personnel` fournis) :
```
static void exporter(List<Personnel> personnels, Path csv) throws IOException
//   trie une COPIE par ANCIENNETÉ DÉCROISSANTE puis NOM CROISSANT :
//     Comparator.comparingInt(Personnel::anciennete).reversed().thenComparing(Personnel::nom)
//   écrit le CSV : 1re ligne en-tête "nom,grade,anciennete", puis une ligne par personnel
//     String.join(",", p.nom(), p.grade().name(), String.valueOf(p.anciennete()))
//   via Files.write(csv, lignes, UTF_8) (lignes = List<String>) ; ne mute PAS la liste source.
```
Idiomatisme (critère) : `Comparator` chaîné (M4), `String.join`, `Files.write` (NIO.2), copie défensive avant tri. **Starter** : corps vide `// TODO`.

**Publics** (≈8) : exporter 3 personnels → relire (`Files.readAllLines`) : en-tête en 1re ligne + 3 lignes triées (ancienneté décroissante) ; format de ligne `nom,GRADE,anciennete` exact ; égalité d'ancienneté départagée par nom alphabétique ; liste vide → fichier avec **seulement l'en-tête** ; la liste source **n'est pas mutée** (ordre d'origine préservé) ; UTF-8 (accents) ; `grade().name()` écrit le nom de la constante enum.
**Privés** (≈8) : un seul personnel → en-tête + 1 ligne ; ancienneté décroissante stricte vérifiée sur 4 éléments ; deux mêmes ancienneté → tri par nom ; écrasement d'un fichier existant (`Files.write` écrase par défaut) ; ancienneté négative triée correctement ; relecture round-trip (export puis `ImportCsv`-like split) cohérente.

### 5.3.3 `parseur-commandes` — split + dispatch + exceptions — critère formateur `idiomatisme`

Classe `ParseurCommandes` (pas de fichier — texte pur) :
```
static int executer(String ligne)
//   découpe ligne.trim() sur les espaces (split("\\s+")) ; 1er token = commande, reste = arguments.
//   dispatch (switch) sur la commande :
//     "somme a b"   -> a + b
//     "diff a b"    -> a - b
//     "produit a b" -> a * b
//     "oppose a"    -> -a
//   - ligne vide / blanche                 -> IllegalArgumentException("commande vide")
//   - commande inconnue                    -> IllegalArgumentException("commande inconnue : <cmd>")
//   - mauvais nombre d'arguments           -> IllegalArgumentException("arguments invalides pour <cmd>")
//   - argument non numérique               -> NumberFormatException (Integer.parseInt, propagée)
```
Idiomatisme (critère) : `switch` lisible sur la commande, découpage propre, validation défensive du nombre d'arguments **avant** conversion. **Starter** : `return 0; // TODO`.

**Publics** (≈8) : `"somme 3 4"` → 7 ; `"diff 10 3"` → 7 ; `"produit 2 5"` → 10 ; `"oppose 8"` → -8 ; commande inconnue (`"racine 9"`) → `IllegalArgumentException` ; argument manquant (`"somme 3"`) → `IllegalArgumentException` ; argument en trop (`"oppose 1 2"`) → `IllegalArgumentException` ; argument non numérique (`"somme a b"`) → `NumberFormatException`.
**Privés** (≈8) : ligne vide/blanche → `IllegalArgumentException` ; espaces multiples entre tokens tolérés (`"somme   3    4"` → 7) ; espaces en tête/queue (`"  oppose 5 "` → -5) ; nombres négatifs en argument (`"somme -3 4"` → 1) ; message « commande inconnue » contient le nom de la commande ; `diff` non commutatif (`"diff 3 10"` → -7) ; `oppose 0` → 0 ; casse sensible (`"SOMME 1 2"` → inconnue).
