## 1. Décisions tranchées au brainstorm (2026-06-03)

1. **On teste des OBJETS** (instanciation + AssertJ), sans console.
2. **Starter : en-têtes structurels FOURNIS, logique à écrire** (décision §10.1, variante validée pour 3.4) : les types structurels (le `record` avec ses composants, l'`enum` avec ses constantes, la `sealed interface` + ses records, l'interface custom) sont **fournis complets** ; le stagiaire écrit la **logique** (méthodes du record, `switch` de l'enum, classe qui implémente l'interface, `switch` exhaustif sur la hiérarchie sealed). Les tests publics compilent dès le départ.
3. **« Comparable custom » = interface de comparaison MAISON, NON générique** : `java.lang.Comparable<T>` utilise les génériques (module 4). On définit donc une interface `Ordonnable { int comparerA(Ordonnable autre); }`, et le contrat porte sur **`comparerA` seul** (renvoie négatif / zéro / positif).
4. **Vérification structurelle : compilation/comportement d'abord, réflexion en DERNIER recours** (§10.2) :
   - **record** : vérifié par les **accesseurs générés** (`coord.x()`) et l'**`equals` généré** (`assertThat(a).isEqualTo(b)` par valeurs) + l'**immuabilité** (une « modification » renvoie un nouvel objet, l'original est inchangé). Pas de réflexion.
   - **enum** : `values()`, `valueOf`, comparaison `==`, méthodes. Pas de réflexion.
   - **sealed** : vérifié par le **`switch` exhaustif sans `default`** (le compilateur garantit l'exhaustivité dans la solution) et le comportement. Pas de `Class.isSealed()`.
5. **`equals`/`hashCode`** : uniquement ceux **générés par les records** ; aucun écrit à la main (module 4).
6. **Nommage** : membres en français, classes `PascalCase`. Pas de format à décimales ici (sauf `getSoldeBase` en `double`, comparé avec tolérance).
7. **Difficulté / durée** : 3.4.1 `moyen` (30) / 3.4.2 `moyen` (30) / 3.4.3 `moyen` (35) / 3.4.4 `difficile` (40).

## 2. Spécification par exercice

`prerequis: []`. Aucune exception. Aucun générique.

### 3.4.1 — coordonnees (`moyen`, 30 min) — **gabarit** — record
- **But** : un `record` immuable, ses accesseurs générés, son `equals` généré, et des méthodes métier (dont une qui renvoie un **nouveau** record).
- **Type FOURNI (en-tête)** : `record Coordonnees(int x, int y)`.
- **API à écrire (corps des méthodes du record)** :
  ```
  record Coordonnees(int x, int y)
    int normeCarree()                       // x*x + y*y
    Coordonnees translater(int dx, int dy)  // renvoie un NOUVEAU Coordonnees(x+dx, y+dy)
  ```
- **Accesseurs** `x()` / `y()` : générés (rien à écrire).
- **Cas publics** : `x()`/`y()` ; `equals` généré (`assertThat(new Coordonnees(1,2)).isEqualTo(new Coordonnees(1,2))`) ; `normeCarree` ; `translater` renvoie les bonnes coordonnées.
- **Cas privés** : **immuabilité** (après `translater`, l'original a toujours ses coordonnées d'origine ; le résultat est un autre objet) ; `equals` faux pour des valeurs différentes ; `normeCarree` avec coordonnées négatives ; `translater` enchaîné.

### 3.4.2 — grade (`moyen`, 30 min) — enum à attributs et méthodes
- **But** : un `enum` avec un attribut (constructeur privé), un accesseur, et une méthode à `switch` exhaustif sur les constantes.
- **Type FOURNI (en-tête + attribut + constructeur + accesseur)** :
  ```
  enum Grade
    SOLDAT(1600), CAPORAL(1800), SERGENT(2200), ADJUDANT(2600), LIEUTENANT(3000)
    private final double soldeBase ; constructeur private ; double getSoldeBase()  // FOURNIS
    String categorie()   // À ÉCRIRE : switch sur this
  ```
- **Contrat `categorie()`** : `SOLDAT, CAPORAL → "Militaire du rang"` ; `SERGENT, ADJUDANT → "Sous-officier"` ; `LIEUTENANT → "Officier"`. `switch` **expression exhaustif sans `default`** (toutes les constantes couvertes).
- **Cas publics** : `Grade.SOLDAT.getSoldeBase()` = 1600 ; `categorie()` de `SOLDAT`/`SERGENT`/`LIEUTENANT` ; `Grade.values().length` = 5.
- **Cas privés** : `getSoldeBase` de `LIEUTENANT` = 3000 ; `categorie()` de `CAPORAL` (« Militaire du rang ») et `ADJUDANT` (« Sous-officier ») ; `Grade.valueOf("SERGENT") == Grade.SERGENT`.

### 3.4.3 — comparable (`moyen`, 35 min) — interface de comparaison custom
- **But** : définir le respect d'un **contrat d'interface** (méthode de comparaison), l'implémenter, et l'exploiter.
- **Type FOURNI** : `interface Ordonnable { int comparerA(Ordonnable autre); }`.
- **API à écrire** :
  ```
  class Dossier implements Ordonnable
    Dossier(String titre, int priorite)
    String getTitre(), int getPriorite()
    @Override int comparerA(Ordonnable autre)   // compare les priorites (Integer.compare)
  class Classement
    static Dossier plusPrioritaire(Dossier[] dossiers)   // priorite max via comparerA, null si vide
  ```
- **Contrat `comparerA`** : renvoie un entier **négatif / nul / positif** selon que la priorité de `this` est inférieure / égale / supérieure à celle de `autre`. Implémentation idiomatique : downcast `(Dossier) autre` puis `Integer.compare(this.priorite, d.priorite)`.
- **Cas publics** : getters ; `comparerA` d'une priorité plus haute → **positif**, plus basse → **négatif** ; `Classement.plusPrioritaire` renvoie le bon dossier.
- **Cas privés** : priorités égales → `comparerA` **nul** ; appel **via le type interface** (`Ordonnable o = new Dossier(...)`) ; `plusPrioritaire` d'un tableau vide → `null` ; `plusPrioritaire` à un seul élément.

### 3.4.4 — evenement (`difficile`, 40 min) — sealed + records + switch exhaustif
- **But** : exploiter une hiérarchie **`sealed`** de records via un **`switch` exhaustif sans `default`** (pattern matching).
- **Types FOURNIS (hiérarchie complète)** :
  ```
  sealed interface Evenement permits Connexion, Deconnexion, Erreur
  record Connexion(String utilisateur) implements Evenement
  record Deconnexion(String utilisateur) implements Evenement
  record Erreur(int code, String message) implements Evenement
  ```
- **API à écrire** :
  ```
  class Journal
    static String resumer(Evenement e)   // switch exhaustif SANS default
  ```
- **Contrat `resumer`** :
  - `Connexion` → `"Connexion de " + utilisateur()`
  - `Deconnexion` → `"Deconnexion de " + utilisateur()`
  - `Erreur` → `"Erreur " + code() + " : " + message()`
  Le `switch` n'a **pas** de `default` : la hiérarchie sealed garantit l'exhaustivité (le compilateur la vérifie).
- **Cas publics** : `resumer` pour chaque type (`Connexion("alice")` → `"Connexion de alice"` ; `Erreur(404, "Not Found")` → `"Erreur 404 : Not Found"`).
- **Cas privés** : `Evenement[]` hétérogène, `resumer` sur chaque élément ; accesseur de record (`new Connexion("bob").utilisateur()` = `"bob"`) ; `equals` généré d'un record (`new Erreur(1,"x").equals(new Erreur(1,"x"))`).

## 7. Hors périmètre (YAGNI)

- **Pas de génériques** (`Comparable<T>`, `List<T>`…) : module 4. L'interface de comparaison est non générique.
- **Pas d'`equals`/`hashCode` manuel** : seulement ceux générés par les records.
- **Pas de collections/streams** (M4) : tableaux uniquement.
- **Pas d'exceptions** (M5) : `plusPrioritaire([])` → `null`.
- **Pas de `default` sur les `switch`** des hiérarchies sealed / enums exhaustifs (anti-pattern ici).
