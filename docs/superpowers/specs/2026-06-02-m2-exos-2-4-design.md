## 1. Décisions tranchées (2026-06-02)

1. **Test par valeur de retour** (comme 2.3) : méthodes `static` à signature imposée, appelées directement par les tests. **Pas de `Scanner`, pas de `main`.**

5. **sommeMatrice** accepte les matrices **rectangulaires** (la somme n'exige pas une matrice carrée).

## 3. Spécification par exercice

### 2.4.1 — factorielle-puissance (`FactoriellePuissance`, `moyen`, 35 min)
Deux méthodes `static` **récursives** :
- `static long factorielle(int n)` — `n ≥ 0`. Cas de base `n <= 1 → 1`. `factorielle(5) = 120`. Type `long` car `13!` dépasse déjà `int`.
- `static long puissance(int base, int exposant)` — `exposant ≥ 0`. Cas de base `exposant == 0 → 1`. `puissance(2, 10) = 1024`.

**Tests publics** : `factorielle(0)=1`, `factorielle(5)=120` ; `puissance(2,10)=1024`, `puissance(5,0)=1`.
**Cas privés** : `factorielle(1)=1`, `factorielle(10)=3628800`, `factorielle(13)=6227020800` (prouve l'intérêt de `long`) ; `puissance(2,0)=1`, `puissance(3,4)=81`, `puissance(7,1)=7`, `puissance(1,100)=1`, `puissance(10,9)=1000000000`.

### 2.4.2 — parcours-matrice-recursif (`ParcoursMatrice`, `difficile`, 45 min)
Une méthode `static` **récursive** :
- `static int sommeMatrice(int[][] matrice)` — somme de tous les éléments, **calculée récursivement**. La matrice peut être **rectangulaire** ; elle a au moins une ligne et une colonne. Le stagiaire peut (et doit, en pratique) ajouter ses propres méthodes `private` récursives auxiliaires.

**Tests publics** : `[[1,2],[3,4]] → 10` ; `[[5]] → 5`.
**Cas privés** : `[[1,2,3],[4,5,6],[7,8,9]] → 45` ; valeurs négatives `[[-1,-2],[-3,-4]] → -10` ; rectangulaire `[[1,2,3],[4,5,6]] → 21` ; une seule ligne `[[2,4,6,8]] → 20`.
