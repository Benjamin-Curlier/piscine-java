## 1. Décisions tranchées au brainstorm (2026-06-02)

1. **Entrée par saisie clavier** (`Scanner(System.in)`), pas de tableau en dur. Permet aux tests privés de varier les cas. Cohérent avec le modèle module 1 (1.3.4).
2. **Test sur la sortie standard** (`CaptureSortie`) + injection de l'entrée (`CaptureEntree`). Le stagiaire écrit **tout dans `main`** ; il n'écrit pas ses propres méthodes (modèle 2.1/2.2 du kickoff).
3. **2.1.4** : rotation **90° horaire**, matrice **carrée n×n** uniquement.
4. **Conventions de sortie figées** (insensibilité locale + cohérence pédagogique) :
   - **Moyenne** : 2 décimales, séparateur **point**, via `String.format(Locale.ROOT, "%.2f", …)`. Une JVM en `fr_FR` afficherait sinon une virgule et casserait les tests.
   - **Indice (2.1.3)** : **base 0**.
5. **Difficulté / durée** : 2.1.1 / 2.1.2 / 2.1.3 = `facile` ; 2.1.4 = `moyen`. Durées : 25 / 20 / 20 / 40 min.

## 2. Convention d'entrée commune (stdin)

Lecture via `Scanner` ; le découpage en lignes n'a pas d'importance pour `Scanner.nextInt()`, mais l'énoncé présente l'entrée comme suit pour le stagiaire :

- **Tableau 1D** (2.1.1, 2.1.2, 2.1.3) :
  - Ligne 1 : `n` (nombre d'éléments, entier ≥ 1).
  - Ligne 2 : `n` entiers séparés par des espaces.
  - 2.1.3 ajoute une **Ligne 3** : la valeur entière recherchée.
- **Matrice carrée** (2.1.4) :
  - Ligne 1 : `n` (côté de la matrice, entier ≥ 1).
  - Lignes suivantes : `n` lignes de `n` entiers (soit `n×n` entiers au total).

Les tests injectent l'entrée via `CaptureEntree`. La graine n'intervient pas (aucun aléatoire dans 2.1).

## 3. Spécification par exercice

### 2.1.1 — min-max-moyenne (`facile`, 25 min)
- **But pédagogique** : parcourir un tableau, accumuler min / max / somme.
- **Entrée** : `n` + n entiers.
- **Sortie** (exactement 3 lignes, `System.lineSeparator()` après chacune) :
  ```
  Min : <min>
  Max : <max>
  Moyenne : <moyenne formatée %.2f, point décimal>
  ```
- **Exemple** : entrée `5` / `3 9 1 7 5` → `Min : 1` / `Max : 9` / `Moyenne : 5.00`.
- **Cas privés** : tableau à 1 élément ; valeurs négatives ; min/max sur les bords ; moyenne non entière (`10 3` → `6.50`) ; grands nombres (pas d'overflow attendu sur la somme : `long` côté solution, à mentionner en correction mais non imposé).

### 2.1.2 — inversion (`facile`, 20 min)
- **But pédagogique** : parcourir un tableau dans l'ordre inverse (ou échange en place).
- **Entrée** : `n` + n entiers.
- **Sortie** : **une ligne** = les éléments dans l'ordre inverse, séparés par **un espace**, suivie d'un retour à la ligne. **Un espace final est toléré** : les tests comparent ligne par ligne après `strip()` (ou découpage sur les espaces), pas à l'octet près. Ne pas pénaliser le débutant pour un espace de séparation final.
- **Exemple** : entrée `4` / `1 2 3 4` → `4 3 2 1`.
- **Cas privés** : tableau à 1 élément (`7` → `7`) ; tableau à 2 éléments ; valeurs négatives ; doublons.

### 2.1.3 — recherche-lineaire (`facile`, 20 min)
- **But pédagogique** : boucle de recherche, arrêt à la première occurrence, gestion du « non trouvé ».
- **Entrée** : `n` + n entiers + valeur cherchée (ligne 3).
- **Sortie** : 
  - si trouvé : `Indice : <i>` où `i` est l'indice **base 0** de la **première** occurrence ;
  - sinon : `Absent`.
- **Exemples** : `3 / 10 20 30 / 20` → `Indice : 1`. `3 / 10 20 30 / 99` → `Absent`.
- **Cas privés** : valeur en première position (`Indice : 0`) ; en dernière ; doublons (renvoie la **première** occurrence) ; valeur absente ; tableau à 1 élément (présent puis absent).

### 2.1.4 — rotation-matrice (`moyen`, 40 min)
- **But pédagogique** : indexation 2D, double boucle, rotation 90° horaire.
- **Entrée** : `n` + n×n entiers.
- **Transformation** : rotation **90° dans le sens horaire**. L'élément `source[i][j]` se retrouve en `resultat[j][n-1-i]`.
- **Sortie** : `n` lignes ; chaque ligne = `n` entiers séparés par un espace, suivie d'un retour à la ligne. **Espace final toléré** (même règle que 2.1.2 : comparaison ligne par ligne après `strip()`).
- **Exemple** : entrée `2` / `1 2` / `3 4` → 
  ```
  3 1
  4 2
  ```
- **Cas privés** : matrice 1×1 (`5` → `5`) ; 3×3 ; valeurs négatives ; vérifier qu'une double rotation (non testée mais mentionnée en correction) ramène… (note pédagogique seulement).

## 4. Structure et constantes (les 4 exos)

Conforme à [`format-exercice.md`](../../format-exercice.md) §2. Chaque dossier `2.1.E-slug/` contient les 11 fichiers, **`tests-prives/` inclus pour les 4**.

- **Package** : `etnc.m2`. **`groupId`** : `etnc.piscine.m2`. **Java 25**.
- **`starter/pom.xml`** : calqué sur module 1 (build-helper ajoute `../tests/src/test/java`). Dépendances JUnit Jupiter + AssertJ.
- **`solution/pom.xml`** : identique + injecte **aussi** `../tests-prives/src/test/java`.
- **Classes** : noms en `PascalCase` dérivés du slug — `MinMaxMoyenne`, `Inversion`, `RechercheLineaire`, `RotationMatrice`. Tests : `<Classe>Test` (public) / `<Classe>PriveTest` (privé).
- **Utilitaires de test** : copier `CaptureSortie` **et** `CaptureEntree` sous `etnc.util` dans `tests/src/test/java/` de chaque exo (autonomie du dossier, comme module 1).

## 5. Énoncés (`sujet.md`)

Structure imposée par `format-exercice.md` §3 : Contexte (touche militaire) / Énoncé / Exemple / Contraintes / Ce qui sera vérifié / Pour aller plus loin. Vouvoiement, phrases courtes, pas d'humour (charte).

- **Contrainte commune à afficher** : « Lisez l'entrée avec `Scanner`. Tout le code peut tenir dans `main`. » + le format d'entrée/sortie exact (l'exemple sert de contrat).

## 7. Ordre d'implémentation

2.1.1 → 2.1.2 → 2.1.3 → 2.1.4 (difficulté croissante). Le premier exo (2.1.1) sert de gabarit : une fois son `pom.xml`, ses utilitaires `etnc.util` et sa structure validés en local, les suivants les réutilisent.

## 8. Hors périmètre (YAGNI)

- Pas de gestion d'entrée malformée (non numérique) : l'énoncé garantit une entrée bien formée ; les tests n'envoient que des entrées valides.
- Pas de matrices rectangulaires (2.1.4 carrée uniquement).
- Pas de méthodes `static` écrites par le stagiaire (réservé à 2.3/2.4).
